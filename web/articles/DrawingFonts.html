SVG fonts are not rendered same on linux and windows. Some of the description below covers how to define a font for an SVG file. The problem is that relative paths are difficult, and it doesn't seem to work if the SVG is embedded into an html file.

Use fallbacks: font-family="Arial, Helvetica, sans-serif"

http://www.w3schools.com/cssref/css_websafe_fonts.asp

HTML font does not seem to be inherited into SVG.

Embedding Font in SVG File

The following works within an SVG file: 
	<?xml version="1.0" standalone="no"?>
	<?xml-stylesheet href="opensans-stylesheet.css" type="text/css"?>
The following works within an SVG file: 
	<style type="text/css"><![CDATA[
	@font-face {
	    font-family: 'open_sansregular';

	    src: url('opensans-regular-webfont.eot');
	    src: url('opensans-regular-webfont.eot?#iefix') format('embedded-opentype'),
	         url('opensans-regular-webfont.woff2') format('woff2'),
	         url('opensans-regular-webfont.woff') format('woff'),
	         url('opensans-regular-webfont.ttf') format('truetype'),
	         url('opensans-regular-webfont.svg#open_sansregular') format('svg');
	    font-weight: normal;
	    font-style: normal;

	}
	]]></style>
